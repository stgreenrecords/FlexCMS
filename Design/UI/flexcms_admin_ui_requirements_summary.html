# FlexCMS Admin UI — Designer Brief & Requirements ## 1. Product Context FlexCMS is an enterprise content management platform with three independent pillars: CMS, DAM, and PIM, all managed through one unified Admin UI. ## 2. Deliverables - Design Tokens / Theme Specification (P0) - Icon Set (~80 icons) (P0) - Component Library (42 components) (P0) - Global Shell / Layout (P0) - 28 Page Mockups across CMS, DAM, PIM, and Shared Admin (P1) - States, Skeletons, and Dark Mode (P2) ## 3. Design Token Structure Requires specific tokens for Surface, Brand, Semantic, Border/Input, and Sidebar colors, plus Radius, Shadow, Typography, and Spacing scales. ## 4. Page Inventory Highlights - **CMS:** Dashboard, Site Manager, Content Tree, Visual Page Editor, etc. - **DAM:** Dashboard, Asset Browser, Asset Detail, Bulk Upload. - **PIM:** Dashboard, Schema Editor, Catalog List, Product Search/Editor, Import Wizard. - **Shared:** Settings, User Management, Theme Editor, Login. ## 5. UI Patterns - Shell / App Frame - Master-Detail - Data Grid Page - Editor Page (Visual WYSIWYG) - Wizard / Stepper - Dashboard ## 6. Core Components - Buttons, Inputs, Badges, Cards (Existing) - Select, Dialogs, Tree View, Data Table, Command Palette, etc. (New) ## 8. Frontend Guidelines (Rules 1-27) Strict adherence to component usage, token-based styling, accessibility, and layout structure (AppShell).
